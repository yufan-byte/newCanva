# Infinite Canvas 新手运行与使用教程

这份教程写给第一次下载项目、不会配置 Python、也不知道 API/ModelScope/ComfyUI/RunningHub 分别是什么的用户。照着做，先把网页打开，再按自己的需求选择一种生成方式即可。

视频教程：

- 无限画布教程：https://www.bilibili.com/video/BV1qvLj67Euh/
- 智能画布教程：https://www.bilibili.com/video/BV1xAVT6hEBk

项目本质上是一个本地网页应用：程序运行在你自己的电脑上，浏览器访问 `http://127.0.0.1:3000/`，也可以访问局域网的地址例如 `http://192.168.10.1:3000/`，局域网内所有人都可以访问这个链接打开这个应用。它不强制绑定某一个平台，你可以选择 API、ModelScope、RunningHub、本地 ComfyUI、火山引擎、即梦 CLI 等方式生成图片、视频或调用大语言模型。

## 一、下载后先做什么

### 1. 先完整解压

下载网盘的最新日期的文件夹，直接双击'启动服务.bat'就可以运行，GitHub下载的版本，需要运行一次'安装依赖.bat'。
mac用户也可以使用，不过由于mac环境与windows不一样，所以需要谷歌搜索opencode，下载安装，无需注册和登录，打开后选择文件夹，直接打字"解决这个程序在mac上的运行环境问题"，之后ai会自行处理。

要使用即梦CLI功能，则需要运行：“安装即梦CLI.bat” 和 “登录即梦CLI.bat” ，运行之后会出现一个网页，打开登录的网址登录之后就可以在API中，点击新增API，将验证协议按键旁边的下拉改为即梦CLI，点击验证地址就会出现绿色的 “✓ 地址验证通过 · 找到 6 个模型，即梦 CLI 已可用，可在画布里选择“即梦 CLI”生成。”


建议路径：

Windows: D:\Infinite-Canvas\
macOS:   /Users/你的用户名/Downloads/Infinite-Canvas/


尽量避免放到路径特别复杂、带很多特殊符号、同步盘权限很严格的位置。

### 2. 看你下载的是哪一种包

如果你是新手，优先下载网盘整包或 Release 稳定版。

### 3. 不要急着配置所有东西

这个项目支持很多生成方式，但你只需要先选一种就能用：

- 想快速体验付费高质量模型：配置推荐 API。（注册之后须要保证账户里面有0.05元钱才能正常验证地址，否则会报错）
- 想免费学习：配置 ModelScope。（需要在账号设置里绑定阿里云并实名认证，否则会报错）
- 已经有本地显卡和 ComfyUI：配置本地 ComfyUI。（可选，没有就使用RunningHub的工作流）
- 想用云端 ComfyUI 工作流：配置 RunningHub。
- 有火山引擎账号：配置火山引擎。
- 有即梦高级会员/积分：安装并登录即梦 CLI。

先跑起来，再慢慢配置。

## 二、Windows 用户怎么运行

### 方法一：网盘整包，一键启动

1. 解压项目文件夹。
2. 打开项目文件夹。
3. 找到下面其中一个文件：

```text
启动服务.bat
```

或：

```text
run.bat
```

4. 双击运行。
5. 黑色命令行窗口会显示访问地址，通常是：

```text
http://127.0.0.1:3000/
```

6. 浏览器一般会自动打开。如果没有自动打开，就手动复制地址到浏览器。

启动成功时，不要关闭黑色命令行窗口。关闭窗口就等于关闭服务。

### 方法二：缺少依赖时先安装依赖

如果双击启动后出现类似这些错误：

```text
ModuleNotFoundError: No module named 'fastapi'
ModuleNotFoundError: No module named 'uvicorn'
ModuleNotFoundError: No module named 'PIL'
```

说明依赖没有装好。回到项目文件夹，双击：

```text
安装依赖.bat
```

这个脚本会优先从 `packages` 文件夹离线安装；离线安装失败时，会尝试联网安装。

安装完成后，再双击：

```text
启动服务.bat
```

### 方法三：没有 python 文件夹怎么办

没有网盘网盘会有的，可以不下载 `python` 文件夹。根目录中已经提供 `python.zip`。

处理方式：

1. 下载 `python.zip`。
2. 解压它。
3. 确认项目目录里出现：

```text
python\python.exe
```

4. 再双击 `启动服务.bat`。

如果没有便携 Python，也可以安装系统 Python。建议安装 Python 3.10 或更高版本，然后运行 `安装依赖.bat`。

### Windows 常见运行问题

#### 1. 双击后窗口一闪而过

通常是启动时报错但窗口关闭太快。解决：

1. 在项目文件夹空白处按住 `Shift`，右键选择“在终端中打开”或“在 PowerShell 中打开”。
2. 输入：

```powershell
.\启动服务.bat
```

3. 看终端最后几行错误。

如果看不懂，把错误复制给 Codex、Trae 或其他 AI 编程助手，让它在项目文件夹里帮你修。

#### 2. 端口 3000 被占用

如果看到类似：

```text
Address already in use
```

说明已经有一个服务占用了 3000 端口。常见原因是你之前已经启动过一次。

处理方式：

1. 找到之前启动的黑色命令行窗口，按 `Ctrl+C` 停止。
2. 或者重启电脑。
3. 再双击启动。

#### 3. 浏览器打不开

手动访问：

```text
http://127.0.0.1:3000/
```

如果你想让同一个局域网里的其他电脑访问，启动窗口里一般会显示一个局域网地址，例如：

```text
http://192.168.x.x:3000/
```

同事电脑打开这个地址即可。前提是你们在同一个网络，且防火墙没有拦截。

#### 4. 防火墙弹窗

Windows 第一次运行可能提示是否允许 Python 或此程序访问网络。建议允许专用网络访问，否则局域网访问、部分 API 请求或 WebSocket 同步可能异常。

## 三、Mac 用户怎么运行

macOS 最大的问题通常不是代码，而是权限和安全限制。

### 1. 先安装 Python

项目需要 Python 3.10 或更高版本。

在终端输入：

```bash
python3 --version
```

如果提示找不到 Python，请去 Python 官网下载安装：

```text
https://www.python.org/downloads/
```

安装后重新打开终端，再检查 `python3 --version`。

### 2. 推荐方式：右键打开启动文件

项目包里可能有以下文件：

```text
mac-启动服务.command
启动服务.command
mac-启动服务.sh
```

优先双击或右键打开 `.command` 文件。

首次运行如果提示：

```text
无法打开，因为无法验证开发者
```

不要直接放弃，按下面做：

1. 右键点击 `mac-启动服务.command` 或 `启动服务.command`。
2. 选择“打开”。
3. 弹窗里再次点“打开”。

以后通常就可以直接双击。

### 3. 如果仍然打不开：先修复权限

项目里一般提供：

```text
mac-修复权限.command
```

右键打开它，等它执行完成，然后再打开启动文件。

如果你想手动在终端执行，可以进入项目目录后运行：

```bash
xattr -r -d com.apple.quarantine . 2>/dev/null
chmod +x *.command 2>/dev/null
chmod +x mac-*.sh 2>/dev/null
```

然后启动：

```bash
./mac-启动服务.command
```

或：

```bash
./mac-启动服务.sh
```

### 4. Mac 依赖安装失败怎么办

如果提示缺少依赖，先运行：

```bash
./mac-安装依赖.sh
```

如果没有这个脚本，可以手动安装：

```bash
python3 -m pip install fastapi uvicorn requests pydantic python-multipart httpx pillow
python3 -m pip install "uvicorn[standard]"
```

安装完成后再启动：

```bash
python3 main.py
```

### 5. Mac 常见报错

#### 报错：Permission denied

说明脚本没有执行权限。

执行：

```bash
chmod +x *.command
chmod +x mac-*.sh
```

#### 报错：Operation not permitted 或无法验证开发者

执行：

```bash
xattr -r -d com.apple.quarantine .
```

或者打开：

```text
系统设置 -> 隐私与安全性 -> 仍要打开
```

#### 报错：python3 command not found

安装 Python 3.10+，然后重新打开终端。

#### 仍然不会处理

建议安装 Codex 或 Trae，打开项目文件夹后直接告诉它：

```text
这个项目在 Mac 上运行报错，请帮我安装依赖并启动。
```

它会根据你的电脑环境继续排查。

## 四、运行成功后下一步做什么

打开网页后，你一般会看到首页、普通无限画布、智能画布、API 设置等入口。

第一件事：先配置至少一种生成方式。

## 五、配置 API 平台

### 1. 推荐 API 的基本配置方式

你自己的运行说明里推荐的 API 注册地址：

```text
https://apimart.ai/register?aff=1uyAbb
```

注册后进入平台后台，复制自己的 API Key。

回到本项目网页：

1. 点击左下角“设置”或“API 设置”。
2. 找到 API 平台配置。
3. 请求地址填写：

```text
https://api.apimart.ai
```

4. 协议选择：

```text
APIMart 异步
```

如果你配置的是其他 OpenAI 兼容平台，协议可以选择 OpenAI。

5. 粘贴 API Key。
6. 点击“验证地址”。
7. 点击“验证协议”。
8. 点击“拉取模型”。
9. 勾选你要用的图片模型、视频模型、LLM 模型。
10. 保存。

如果模型被自动分错类，例如图片模型被分到视频里，可以手动删除再添加到正确分类。

### 2. 为什么推荐异步协议

普通 OpenAI 协议一般是直接请求，网络中断或等待超时可能导致任务失败。

异步协议是先把任务提交到后台，后台生成成功后再把结果返回。它更适合图片、视频这种耗时任务，网络波动时也更稳。

### 3. API 配置后仍然不能拉取模型

检查这些点：

- API Key 是否复制完整。
- 请求地址是否多了空格。
- 账户里是否有余额。视频字幕里提到有些平台至少需要有少量余额才能正常验证和拉取。
- 协议是否选对。APIMart 用 APIMart 异步，其他 OpenAI 兼容平台多半用 OpenAI。
- 网络是否能访问对应平台。

### 4. 不要把 API Key 发给别人

项目的真实密钥一般保存在本地配置里。不要把 `API/.env` 或任何包含 Key 的文件发到 GitHub、群聊、网盘公开目录。

## 六、配置 ModelScope 免费生成

ModelScope 适合新手免费学习。它支持一些免费的图片模型、LLM 模型，也可以调用 LoRA。

基本步骤：

1. 注册 ModelScope。
2. 在 ModelScope 账号里创建长期 Token。
3. 回到项目网页，打开 API 设置。
4. 找到 ModelScope。
5. 填入 Token。
6. 点击验证。
7. 点击拉取模型。
8. 勾选想用的图片模型、LLM 模型。
9. 保存。

使用建议：

- 生成图片可以从 ModelScope 节点开始。
- 大语言模型要选对应的 LLM 模型。
- 如果要识别图片，必须选择 VL/视觉模型。
- ModelScope 免费适合学习，但速度和分辨率可能不如付费 API。

### ModelScope LoRA 怎么用

1. 在 ModelScope 找到你想用的 LoRA。
2. 复制 LoRA 链接或模型信息。
3. 回到项目 API 设置。
4. 在 ModelScope 的 LoRA 区域点击添加。
5. 粘贴链接。
6. 选择对应基础模型。
7. 保存。

之后在 ModelScope 生成节点里可以选择 LoRA。

## 七、配置本地 ComfyUI

如果你电脑有较好的显卡，并且已经会使用 ComfyUI，可以连接本地 ComfyUI。

### 1. 什么时候适合用本地 ComfyUI

适合：

- 有 16GB 以上显存，最好 4090 或更强。
- 已经安装好 ComfyUI。
- 想运行自己的工作流。
- 想完全本地免费生成。

不适合：

- 没有独立显卡。
- 显存很小。
- 不想折腾模型和节点。

这种情况建议先用 API、ModelScope 或 RunningHub。

### 2. 设置 ComfyUI 地址

1. 先启动你本地的 ComfyUI。
2. 确认 ComfyUI 能打开，例如：

```text
http://127.0.0.1:8188/
```

3. 回到 Infinite Canvas。
4. 打开左下角设置。
5. 找到 ComfyUI 设置。
6. 填入 ComfyUI 后端地址。
7. 保存。

如果你有多台电脑或多张显卡，也可以配置局域网 ComfyUI 地址。

### 3. 导入自定义 ComfyUI 工作流

先在 ComfyUI 里确认工作流能正常跑。

然后：

1. 在 ComfyUI 中点击导出 API 格式工作流。通常是“Export API”或“导出为 API”。
2. 回到项目的 ComfyUI 设置。
3. 上传工作流 JSON。
4. 项目会识别工作流里的图片、文本、LoRA、seed、分辨率等参数。
5. 勾选你希望暴露给画布使用的输入。
6. 调整顺序，例如图 1、图 2、提示词。
7. 在内置测试区上传测试图片和提示词。
8. 点击运行测试。
9. 确认结果没问题后保存配置。

之后在普通画布或智能画布里，新建 ComfyUI 节点，选择“自定义”，就能调用这个工作流。

### 4. 工作流输入顺序一定要注意

如果你的工作流需要两张图，例如：

- 图 1：原图
- 图 2：风格参考图或 mask 图

导入时一定要确认编号和顺序。图 1、图 2 对错了，结果会完全不对。

如果工作流里有 mask，RunningHub 或 API 不一定能直接接收 mask 类型。可以把 mask 转成黑白图片，再作为普通图片输入。

## 八、配置 RunningHub

RunningHub 适合没有本地高显卡、但想用 ComfyUI 工作流的人。它可以调用云端工作流、AI 应用和收费模型。

### 1. 填 Key

1. 打开 API 设置。
2. 找到 RunningHub。
3. 点击获取 Key。
4. 复制 Key。
5. 粘贴到项目里。
6. 保存。

字幕里提到 RunningHub 可能有两类消耗方式：

- 点数/积分：适合工作流和部分 AI 应用。
- 账户余额：部分第三方模型，例如 GPT、Nano Banana 之类，可能必须用余额。

具体以 RunningHub 页面显示为准。

### 2. 添加 AI 应用或工作流

AI 应用：

1. 在 RunningHub 找到 AI 应用。
2. 点击 API。
3. 复制 API 信息。
4. 回到项目粘贴并创建。

工作流：

1. 在 RunningHub 找到工作流。
2. 点击 API。
3. 复制 API 信息。
4. 回到项目粘贴并创建。

### 3. 工作流建议先简化

如果别人的 RunningHub 工作流特别复杂，建议先删掉无关节点，只保留必要输入和最终结果图。这样接入 Infinite Canvas 时更稳定，也更容易给团队成员使用。

## 九、火山引擎和即梦 CLI

### 火山引擎

项目内已经有火山引擎入口。通常需要填写：

- API Key
- Access Key ID
- Secret Access Key
- Project Name
- Region

填好后拉取模型即可。视频里提到火山引擎的部分人脸认证或审核问题可能还会卡住，遇到这种情况可以换其他 API 或使用 RunningHub/即梦。

### 即梦 CLI

如果你有即梦高级会员或积分，可以使用项目里的即梦 CLI 入口。

Windows 包里可能有：

```text
安装即梦CLI.bat
登录即梦CLI.bat
```

大致流程：

1. 双击安装即梦 CLI。
2. 双击登录即梦 CLI。
3. 按命令行提示完成登录。
4. 回到项目里选择即梦相关模型。

即梦 CLI 适合调用自己的即梦权益，支持文生图、图生图、文生视频、图生视频等。具体能力以你当前即梦账号和项目版本为准。

## 十、普通无限画布怎么用

普通无限画布更像节点编辑器，适合把图片、提示词、生成节点、放大节点、视频节点、LLM 节点串起来。

常用节点：

- 图片节点：上传、拖入、粘贴图片。
- 提示词节点：写正向提示词，也可以打开提示词库。
- API 生成节点：调用 API 平台生成图片或视频。
- ModelScope 节点：调用 ModelScope 免费模型。
- ComfyUI 节点：调用本地或自定义工作流。
- RunningHub 节点：调用云端工作流或 AI 应用。
- LLM 节点：调用大语言模型，可以处理文本，也可以识别图片。
- 循环节点：批量生成多张图，例如 10 个卖点图、多个分镜、批量扩图。
- 输出节点：收集结果，方便预览、下载、继续连接。

### 基本操作

- 拖动画布空白处：移动视野。
- 滚轮：缩放画布。
- 框选多个节点后按 `Ctrl+G`：成组。
- `Ctrl+Shift+G`：取消分组。
- 节点之间拖线：连接输入输出。
- 从输出图拖出来：可以变成新的图片节点或继续接下一个节点。
- 右键节点：打开快捷操作。
- 双击图片：预览或进入图片编辑。
- 上方或右上角日志：查看运行错误。

### 图片编辑功能

双击图片后可以使用：

- 预览、缩放、拖动查看。
- 对比原图和结果图。
- 裁剪。
- 遮罩编辑：只让 AI 修改涂抹区域。
- 画笔局部编辑。
- 扩图/outpaint：向外扩展画面。
- 网格切图：把九宫格、多宫格图片拆成单张。
- 视频帧抽取：从视频里提取关键帧。

### 循环节点适合做什么

循环节点适合批量任务，例如：

- 一次生成多个卖点图。
- 对一组分镜逐张扩图。
- 同一个人物批量生成不同表情。
- 用同一个工作流批量跑多张图。

逻辑是：上游输入一个基础提示词，下方写多个变化项，循环节点每运行一次就自动切换到第几个变化项。

## 十一、智能画布怎么用

智能画布更像卡片式创作工具，不需要你手动拉很多线。它把 API、视频、ModelScope、ComfyUI、RunningHub 等能力整合在卡片里。

适合：

- 不想学习复杂节点连接。
- 想快速做灵感板。
- 想直接编辑某一张图。
- 想用素材库和 `@` 引用图片。
- 想给团队成员降低操作门槛。

### 智能画布基本操作

1. 新建智能画布。
2. 拖入图片、视频或素材。
3. 选择要用的生成方式，例如 API、ModelScope、ComfyUI、RunningHub。
4. 写提示词。
5. 点击运行。

如果是编辑模式，看到哪张图就编辑哪张图，不需要手动连接一堆节点。

### 智能画布循环节点

智能画布里的循环节点更直观：

- 把循环节点拖到目标卡片附近，会自动连接。
- 连接成功后通常会显示高亮状态。
- 设置循环次数。
- 写多条变化提示词。
- 点击运行即可批量生成。

### 全局预览

字幕里提到可以按 `Z` 进行全局节点预览。大量节点时也能快速查看整体结构。

## 十二、素材库怎么用

素材库是全局的，不只属于某一个画布。你在一个画布里保存的素材，其他画布也可以用。

常见用途：

- 保存角色图。
- 保存场景图。
- 保存风格参考。
- 保存视频素材。
- 在智能画布里通过 `@` 调用素材。

### 打开素材库

普通画布和智能画布里一般都有“素材库/资产库”按钮。智能画布中也可以按 `A` 打开素材库。

### 上传素材

可以：

- 直接拖图片进去。
- 上传视频。
- 从画布输出结果保存到素材库。
- 重命名素材。
- 按角色、场景、参考图等分类管理。

### 在提示词里引用素材

在智能画布中，可以输入 `@`，选择素材库里的图片。这样不需要手动连线，也能把图片作为参考输入。

## 十三、视频生成注意事项

项目支持视频生成、图生视频、视频转视频等能力，但不同平台限制不同。

特别注意：很多 API 平台不接受你直接上传本地视频文件路径。它们需要公网可访问的视频 URL。

所以如果你在视频节点里使用本地视频，可能需要先点：

```text
上传云端
```

上传成功后会得到一个可访问地址，再点击运行。

如果上传云端失败，可以：

- 使用自己的阿里云/对象存储。
- 使用其他能生成公网 URL 的文件服务。
- 手动把公网视频链接粘贴到节点里。

## 十四、更新项目怎么做

### 新手推荐：重新下载最新整包

如果你不会处理 Git，最稳妥的方式是：

1. 下载最新版本。
2. 解压到新文件夹。
3. 把旧项目里的个人数据复制过去。

常见需要保留的个人数据：

```text
API/
data/
assets/
output/
workflows/custom/
```

这些通常包含你的 API 配置、画布、素材库、生成结果、自定义工作流。

不要直接把旧版本整个覆盖到新版本上，尤其是 `static` 目录结构变过时，可能导致页面打不开。


## 十五、浏览器缓存问题

更新代码后，如果页面还是旧样式、按钮不见了、智能画布组件没出来，通常是浏览器缓存。

先试：

```text
Ctrl + Shift + R
```

如果还不行：

1. 按 `F12` 打开开发者工具。
2. 找到刷新按钮。
3. 右键刷新按钮。

或者换一个浏览器测试。

## 十六、常见问题解答

### Q1：这个项目必须联网吗？

打开本地网页不一定需要联网。但如果你要调用 API、ModelScope、RunningHub、火山引擎、即梦等服务，就需要联网。

如果你只调用本地 ComfyUI，理论上可以离线，但你本地模型和依赖必须已经准备好。

### Q2：没有显卡能用吗？

能用。你可以用 API、ModelScope、RunningHub 等云端方式。没有显卡不建议折腾本地 ComfyUI 大模型。

### Q3：必须配置 ComfyUI 吗？

不必须。API、ModelScope、RunningHub 三种方式任选一种就能用。

### Q4：免费用户怎么开始？

建议先配置 ModelScope，体验免费图片模型和 LLM 模型。等理解流程后，再配置付费 API 或 RunningHub。

### Q5：API 生成为什么报错？

常见原因：

- Key 错了。
- 请求地址错了。
- 协议选错了。
- 账户余额不足。
- 模型被平台下架或改名。
- 图片或视频不符合平台审核要求。
- 本地视频没有公网 URL。

先打开右上角日志，看具体错误。

### Q6：为什么视频生成提示不能上传本地文件？

很多平台只接受公网 URL。先使用“上传云端”，或手动提供公网视频链接。

### Q7：为什么图片识别不准？

如果使用 ModelScope 识别图片，要选 VL/视觉模型。普通文本模型不能看图。

### Q8：为什么 RunningHub 工作流跑不起来？

检查：

- Key 是否正确。
- 使用的是点数还是余额。
- 工作流是否发布。
- API 信息是否复制完整。
- 必填图片是否都传了。
- 工作流输入顺序是否和项目里配置的一致。

### Q9：为什么更新后页面打不开？

可能是旧版本目录结构和新版本不兼容。建议下载最新整包，然后只复制个人数据目录，不要混合覆盖 `static`。

### Q10：我完全不会看报错怎么办？

推荐安装 Codex 或 Trae。

做法：

1. 打开 Codex 或 Trae。
2. 选择这个项目文件夹。
3. 直接输入：

```text
这个项目运行报错，请帮我修到能打开 http://127.0.0.1:3000/
```

它会读取报错和项目文件，帮你安装依赖或修改启动问题。

## 十七、推荐新手路线

如果你是第一次使用，按这个顺序最稳：

1. 解压项目。
2. Windows 双击 `启动服务.bat`；Mac 右键打开 `mac-启动服务.command`。
3. 浏览器打开 `http://127.0.0.1:3000/`。
4. 先配置推荐 API 或 ModelScope。
5. 新建智能画布。
6. 拖入一张图片。
7. 写一句简单提示词。
8. 点击运行。
9. 学会保存素材到素材库。
10. 再学习普通画布的节点连接、循环节点、ComfyUI、RunningHub。

不要一开始就把所有平台都配置一遍。能生成第一张图，才是最重要的第一步。

## 十八、功能总览

当前项目大致支持：

- 普通无限画布：节点式连接，适合复杂工作流。
- 智能画布：卡片式创作，适合新手和灵感板。
- API 生图：支持 OpenAI 兼容协议、APIMart 异步协议等。
- API 视频：支持多种视频模型，具体以平台模型列表为准。
- ModelScope：免费图片模型、LLM 模型、VL 视觉模型、LoRA。
- RunningHub：云端工作流、AI 应用、收费模型调用。
- 本地 ComfyUI：调用本机或局域网 ComfyUI。
- 自定义 ComfyUI 工作流：导出 API JSON 后导入项目。
- 火山引擎：图片/视频模型调用。
- 即梦 CLI：调用即梦账号权益。
- LLM 节点：文本对话、图片理解、提示词反推。
- 图片编辑：裁剪、遮罩、局部编辑、扩图、对比预览。
- 多宫格切图：适合分镜、卖点图、角色参考图拆分。
- 循环节点：批量生成、并发生成、批量扩图。
- 素材库：图片/视频资产管理，跨画布复用。
- 提示词库：保存常用提示词模板。
- 输出节点：收集、预览、下载结果。
- 实时同步：多个窗口打开同一画布时可以同步变化。
- 日志查看：运行失败时快速定位问题。
- 中英文切换、深浅色模式。
